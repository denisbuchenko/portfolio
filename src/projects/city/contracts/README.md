# City Contracts (архитектурные интерфейсы)

Здесь лежат **только** `type`/`interface` и комментарии — **без реализации** и без привязки к стеку.
Цель: заранее зафиксировать зоны ответственности и границы модулей для проекта **City**.

## Как ориентироваться

- `core/` — базовые типы (идентификаторы, трансформы, жизненный цикл, события)
- `world/` — создание мира и доступ к объектам мира
- `character/` — создание персонажа, управление, движение, анимации
- `npc/` — создание NPC, поведение и анимации NPC
- `physics/` — столкновения/проверки пересечений (контракты)
- `camera/` — камера и её анимация/риги
- `transforms/` — изменение трансформов объектов мира
- `navigation/` — логика “достиг позиции”

## Базовые принципы

- Контракты должны быть **тестируемыми** и **заменяемыми**: реализация подменяется другим модулем без правок вызывающего кода.
- Данные “наружу” — простые структуры (`Vec3`, `Transform`), без ссылок на конкретные библиотеки.
- Логика разделяется на **создание** (Factory), **состояние** (модели/handles) и **системы** (update/tick).
