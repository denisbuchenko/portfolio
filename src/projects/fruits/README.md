# Проект "Фрукты"

Проект для рендера 3D моделей фруктов с использованием Three.js. Поддерживает создание 7 независимых слоёв (bits=1..7) с настраиваемыми пресетами, оптимизированный рендер через InstancedMesh и интеграцию в другие проекты (например, пазлы).

## Архитектура

Проект разделён на модули с чётким разделением ответственности:

```
fruits/
├── foodCatalog.ts      # Парсинг 3D моделей из glTF
├── types.ts            # Типы конфигурации
├── utils.ts             # Утилиты (математика, цвета, фильтрация)
├── instancing.ts        # Логика InstancedMesh (оптимизация)
├── placement.ts         # Размещение объектов без пересечений
├── fruitRenderer.ts     # Главный рендерер (ядро логики)
├── sceneComposer.ts     # Компоновщик сцены (высокоуровневый API)
├── renderer.ts          # Технический слой WebGL
├── ui.ts                # UI компоненты
└── FruitsProject.ts     # Точка входа (связывает всё вместе)
```

### Поток данных

```
FruitsProject.ts
    ↓
sceneComposer.ts (обёртка над fruitRenderer)
    ↓
fruitRenderer.ts (ядро)
    ├→ foodCatalog.ts (загрузка моделей)
    ├→ instancing.ts (создание InstancedMesh)
    ├→ placement.ts (размещение объектов)
    ├→ utils.ts (утилиты)
    └→ renderer.ts (WebGL настройки)
```

## Описание файлов

### `foodCatalog.ts` — Парсинг 3D моделей

**Назначение:** Загружает и парсит glTF файл с 3D моделями фруктов, подготавливает их для использования.

**Что делает:**
1. Загружает glTF через `GLTFLoader`
2. Находит все базовые ноды (без underscore в имени) — это корневые узлы для каждого типа фрукта
3. Фильтрует мусорные ноды (RootNode, FBX-обёртки) — проверяет наличие реальных мешей вида `${base}_${base}_0`
4. Настраивает материалы: конвертирует в `MeshBasicMaterial` (unlit) для чистого цвета из текстуры
5. Центрирует и нормализует размер каждого объекта (чтобы maxDim = 1)

**Экспортирует:**
- `FoodEntry` — тип записи о фрукте (name, object, normalizedScale)
- `loadFoodCatalog(gltfUrl)` — функция загрузки

**Используется в:** `fruitRenderer.ts`

---

### `types.ts` — Типы конфигурации

**Назначение:** Определяет все типы для конфигурации рендера фруктов.

**Содержит:**
- `FruitLayerBits` — тип для bits-слоя (1 | 2 | 3 | 4 | 5 | 6 | 7)
- `FruitLayerPreset` — настройки одного слоя (фон, направление, скорость, размеры, набор фруктов)
- `FruitBackgroundPresetsConfig` — полная конфигурация (камера, освещение, движение, 7 слоёв)

**Используется в:** `fruitRenderer.ts`, `sceneComposer.ts`, `puzzleRenderer.ts`

---

### `utils.ts` — Утилиты

**Назначение:** Вспомогательные функции для работы с фруктами.

**Содержит:**
- `clamp(v, a, b)` — ограничение значения между min и max
- `norm2(v)` — нормализация 2D вектора
- `rand01(seed)` — детерминированный генератор случайных чисел (0..1)
- `hexToRgb8(hex)` — конвертация hex цвета в RGB (0..255)
- `createSolidTexture(hex)` — создание простой текстуры из цвета (для fallback)
- `patchMaterialForBackground(mat)` — настройка Basic материала (unlit)
- `filterCatalogEntries(entries, include?, exclude?)` — фильтрация записей каталога
- `pickUnique(entries, count, seed)` — выбор уникальных записей (Fisher-Yates)

**Используется в:** `fruitRenderer.ts`, `instancing.ts`, `placement.ts`

---

### `instancing.ts` — Логика InstancedMesh

**Назначение:** Создание и управление InstancedMesh для оптимизации рендера. Вместо создания отдельных Group для каждого фрукта, используем InstancedMesh с общей геометрией и материалами.

**Что делает:**
1. **`buildTypeDefs(entries)`** — собирает определения типов:
   - Преобразует Group в плоский список геометрий с материалами
   - Применяет трансформации для правильного позиционирования в координатах корня группы
   - Возвращает `Map<name, TypeDef>` где TypeDef содержит геометрию и материалы

2. **`assignInstancesToTypes(picked, count, seed)`** — распределяет инстансы по типам:
   - Детерминированно назначает каждый инстанс одному из picked типов
   - Возвращает массив типов и Map с количеством инстансов каждого типа

3. **`createTypeLayersForBits(bits, defs, counts)`** — создаёт TypeLayer для bits-слоя:
   - Для каждого типа создаёт InstancedMesh с нужным количеством инстансов
   - Настраивает layers для фильтрации по bits
   - Возвращает `Map<name, TypeLayer>`

**Типы:**
- `TypeDef` — определение типа (геометрия + материалы)
- `TypeLayer` — слой инстансов для типа (меши, количество, baseScale)

**Используется в:** `fruitRenderer.ts`

---

### `placement.ts` — Размещение объектов

**Назначение:** Логика размещения объектов на экране без пересечений, используя spatial hash для быстрой проверки коллизий.

**Что делает:**
1. **`getPlacementState(...)`** — создаёт или возвращает кэшированное состояние размещения:
   - Вычисляет размер сетки клеток на основе количества инстансов и аспекта экрана
   - Создаёт сетку клеток и детерминированно перемешивает её (убирает "равномерную матрицу")
   - Настраивает spatial hash (размер клетки примерно под max size)
   - Кэширует состояние по ключу `${bits}_${w}_${h}_${margin}`

2. **`tryPlace(st, seed, r, chaos)`** — пытается разместить объект без пересечений:
   - Берёт следующую клетку из перемешанной сетки
   - Применяет jitter (широкий, но ограниченный) для "непредсказуемости"
   - Проверяет коллизии через spatial hash (проверяет соседние клетки)
   - Если находит свободное место — размещает и возвращает позицию
   - Если не находит за 22 попытки — fallback на центр следующей клетки

3. **`clearPlacementCache()`** — очищает кэш состояний (вызывается при resize)

**Алгоритм spatial hash:**
- Разбивает экран на клетки размера `hashCellSize`
- Для каждой размещённой позиции добавляет ID в соответствующую клетку и соседние
- При проверке коллизий проверяет только соседние клетки (быстро)

**Используется в:** `fruitRenderer.ts`

---

### `fruitRenderer.ts` — Главный рендерер

**Назначение:** Ядро логики рендера фруктов. Управляет загрузкой, анимацией и рендером всех фруктов в 7 слоях.

**Архитектура:**
- Использует `InstancedMesh` для оптимизации (общая геометрия, отдельные матрицы)
- Рендерит в 7 отдельных `RenderTarget` для каждого bits-слоя (для пазлов)
- Поддерживает `PerspectiveCamera` для объёмного вида
- Использует spatial hash для размещения без пересечений

**Основные функции:**

1. **`load()`** — асинхронная загрузка:
   - Загружает каталог через `loadFoodCatalog`
   - Собирает определения типов через `buildTypeDefs`
   - Для каждого bits-слоя (1..7):
     - Фильтрует и выбирает типы фруктов
     - Распределяет инстансы по типам
     - Создаёт TypeLayer'ы и добавляет меши в сцену
     - Создаёт инстансы с детерминированными параметрами (seed, axis, angVel, etc.)

2. **`resize(w, h, dpr)`** — обновление размеров:
   - Создаёт/обновляет RenderTarget'ы с учётом `rtScale`
   - Настраивает PerspectiveCamera (fov, позиция, lookAt)
   - Вычисляет глубину распределения по Z
   - Сбрасывает состояние размещения

3. **`update(timeSec, dpr)`** — обновление анимации:
   - Вычисляет delta time (ограничен для стабильности)
   - Для каждого инстанса:
     - Инициализирует позицию (если ещё не инициализирован) через `getPlacementState` и `tryPlace`
     - Обновляет движение по направлению
     - Применяет покачивание (sway) перпендикулярно направлению
     - Обрабатывает wrap (зацикливание при выходе за границы)
     - Вычисляет масштаб
     - Обновляет 3D вращение вокруг случайной оси (tumble)
     - Обновляет матрицу инстанса через `setMatrixAt`
   - Управляет частотой рендера (если `updateFps > 0`)

4. **`renderTargets(renderer)`** — рендер всех слоёв в offscreen RenderTarget'ы:
   - Для каждого bits-слоя (1..7):
     - Устанавливает RenderTarget
     - Очищает цветом фона слоя
     - Устанавливает camera.layers для фильтрации
     - Обновляет матрицы инстансов через `flushInstanceMatrices`
     - Рендерит сцену

5. **`renderLayerToScreen(renderer, bits)`** — рендер конкретного слоя на экран:
   - Аналогично `renderTargets`, но рендерит напрямую на экран (для превью)

6. **`flushInstanceMatrices(bits)`** — помечает матрицы инстансов как "грязные":
   - Для всех TypeLayer'ов нужного bits-слоя устанавливает `instanceMatrix.needsUpdate = true`
   - Сбрасывает флаг `_dirty`

7. **`dispose()`** — освобождает ресурсы:
   - Удаляет RenderTarget'ы
   - Удаляет fallback текстуры
   - Удаляет геометрию и материалы (с дедупликацией)

**Внутренние типы:**
- `FruitInstance` — данные одного инстанса (позиция, вращение, скорость, etc.)

**Используется в:** `sceneComposer.ts`, `puzzleRenderer.ts`

### `sceneComposer.ts` — Компоновщик сцены

**Назначение:** Высокоуровневый API для работы с фруктами. Обёртка над `fruitRenderer` с удобными методами.

**Что делает:**
- Создаёт `FruitBackgroundRenderer` через `createFruitBackgroundRenderer`
- Предоставляет простой интерфейс: `load()`, `resize()`, `update()`, `renderLayerToScreen()`, `getLayerTexture()`
- Скрывает детали реализации от пользователя

**Используется в:** `FruitsProject.ts`

---

### `renderer.ts` — Технический слой WebGL

**Назначение:** Создание и настройка WebGLRenderer, управление resize.

**Что делает:**
1. **`createFruitsRenderer(canvas, settings?)`** — создаёт WebGLRenderer:
   - Настраивает параметры (alpha, antialias, depth, etc.)
   - Устанавливает `outputColorSpace = SRGB` для корректных цветов
   - Устанавливает `setPixelRatio(1)` (управляем размером вручную)

2. **`resizeRenderer(canvas, renderer, getDpr)`** — обновляет размеры:
   - Вычисляет размеры с учётом DPR
   - Обновляет canvas.width/height
   - Вызывает `renderer.setSize()`

**Используется в:** `FruitsProject.ts`

---

### `ui.ts` — UI компоненты

**Назначение:** Создание DOM структуры для проекта фруктов.

**Что делает:**
- Создаёт HTML структуру (canvas, панель, статус)
- Возвращает ссылки на canvas и statusEl

**Используется в:** `FruitsProject.ts`

---

### `FruitsProject.ts` — Точка входа

**Назначение:** Связывает все модули вместе, запускает рендер-луп.

**Что делает:**
1. Создаёт UI через `createFruitsUI`
2. Создаёт WebGL рендер через `createFruitsRenderer`
3. Создаёт компоновщик сцены через `createSceneComposer`
4. Загружает модели через `scene.load()`
5. Запускает рендер-луп:
   - Обновляет resize при изменении размеров
   - Автопереключает пресеты каждые 2.4 сек (для превью)
   - Обновляет анимацию через `scene.update()`
   - Рендерит слой через `scene.renderLayerToScreen()`
   - Обновляет статус

**Используется в:** `main.ts` (точка входа приложения)

## Взаимодействие модулей

### Последовательность инициализации:

```
1. FruitsProject.ts создаёт UI и рендер
2. FruitsProject.ts создаёт sceneComposer
3. sceneComposer создаёт fruitRenderer
4. fruitRenderer.load():
   a. foodCatalog.loadFoodCatalog() → загружает glTF
   b. instancing.buildTypeDefs() → собирает геометрию
   c. instancing.assignInstancesToTypes() → распределяет инстансы
   d. instancing.createTypeLayersForBits() → создаёт InstancedMesh
5. fruitRenderer.resize() → настраивает камеру и RenderTarget'ы
6. Рендер-луп:
   a. fruitRenderer.update() → обновляет анимацию
      - placement.getPlacementState() → получает состояние размещения
      - placement.tryPlace() → размещает объекты
   b. fruitRenderer.renderLayerToScreen() → рендерит на экран
```

### Зависимости:

- `FruitsProject.ts` → `sceneComposer.ts`, `renderer.ts`, `ui.ts`
- `sceneComposer.ts` → `fruitRenderer.ts`, `types.ts`
- `fruitRenderer.ts` → `foodCatalog.ts`, `instancing.ts`, `placement.ts`, `utils.ts`, `types.ts`
- `instancing.ts` → `utils.ts`
- `placement.ts` → `utils.ts`, `types.ts`

## Ключевые концепции

### InstancedMesh оптимизация

Вместо создания отдельных `THREE.Group` для каждого фрукта (дорого по памяти и drawcalls), используем `THREE.InstancedMesh`:
- Общая геометрия и материалы для всех инстансов одного типа
- Отдельная матрица для каждого инстанса (обновляется каждый кадр)
- Значительно меньше памяти и drawcalls

### Spatial Hash для размещения

Для быстрой проверки коллизий при размещении объектов:
- Разбиваем экран на клетки размера `hashCellSize`
- Для каждой размещённой позиции добавляем ID в соответствующую клетку
- При проверке коллизий проверяем только соседние клетки (O(1) вместо O(n))

### 7 слоёв (bits=1..7)

Каждый слой соответствует комбинации RGB каналов в paint-маске пазла:
- bits=1 → только R канал
- bits=2 → только G канал
- bits=3 → R+G каналы
- bits=4 → только B канал
- bits=5 → R+B каналы
- bits=6 → G+B каналы
- bits=7 → R+G+B каналы

Каждый слой имеет свой набор фруктов, направление движения, скорость, размеры и цвет фона.

### PerspectiveCamera для объёма

Используется `PerspectiveCamera` вместо `OrthographicCamera` для объёмного вида:
- FOV настраивается через `config.camera.fovDeg`
- Глубина распределения по Z через `config.camera.depthCssPx`
- Камера позиционируется так, чтобы на плоскости z=0 видимая высота была равна h (в пикселях)

## Использование в других проектах

Проект фруктов используется в пазлах (`puzzleRenderer.ts`):
- Импортирует `createFruitBackgroundRenderer` и `FruitBackgroundPresetsConfig`
- Создаёт рендерер с конфигурацией из `CONFIG.puzzle.background3d`
- Рендерит фрукты в offscreen RenderTarget'ы перед основным кадром
- Использует текстуры слоёв как фон вместо сплошного цвета

---

## Настройка

Все настройки находятся в `CONFIG.puzzle.background3d`:
- `instanceMul` — множитель количества объектов
- `sizeMul` — множитель размера
- `positionChaos` — беспорядочность позиций (0..1)
- `camera.fovDeg` — поле зрения камеры
- `camera.depthCssPx` — глубина по Z
- `motion.*` — параметры движения
- `layers[1..7]` — настройки каждого слоя
