Техническое задание
Проект: Интерактивная система частиц с двумя режимами аттракторов

1. Цель
Разработать веб-приложение с визуализацией не менее 1000 частиц, поддерживающее два режима взаимодействия:

Режим 1: Частицы отталкиваются от курсора пользователя с эффектом вихревого движения на границах зоны влияния. Аттрактор динамически следует за положением мыши.
Режим 2: Частицы, попадающие в зону захвата предопределённого пути (кривая Безье или кубический сплайн), перемещаются вдоль траектории до её завершения, после чего возвращаются в свободное движение.
Переключение между режимами осуществляется через две кнопки в интерфейсе.
2. Технические требования

2.1. Производительность

Обеспечить стабильную частоту кадров (60 FPS) при количестве частиц 1000+ на устройствах среднего уровня (включая мобильные).
Максимально задействовать GPU для расчётов движения частиц. Запрещено полное управление позициями частиц на CPU в основном цикле рендеринга.
2.2. Технологический стек

Основной фреймворк: Three.js (версия не ниже 0.160) с использованием WebGLRenderer.
Альтернативный вариант: PixiJS v8, только если Three.js не обеспечивает необходимой производительности для сплайновых траекторий.
Шейдеры: GLSL ES 3.0 для vertex и fragment шейдеров.
2.3. Режим 1: Аттрактор-мышь с вихрем

Данные:
Положение курсора передаётся в шейдеры как uniform-переменная в мировых координатах.
Радиус зоны влияния: 1.5 единиц виртуального пространства.
Логика:
Внутри зоны влияния:
Отталкивание от центра пропорционально расстоянию до курсора.
Вихревое движение на границах зоны, рассчитываемое через векторное произведение направления к курсору и оси Z.
Вне зоны: базовое хаотичное движение частиц (на основе шума или предопределённых траекторий).
2.4. Режим 2: Сплайновый путь

Формат пути:
Кривая Безье третьего порядка, задаваемая четырьмя контрольными точками (P0, P1, P2, P3).
Данные пути передаются в шейдеры через uniform-массив.
Захват частиц:
Гибридный подход:
На CPU периодически (каждые 5 кадров) рассчитывается расстояние частиц до пути (аппроксимация через 30 отрезков).
При попадании в зону захвата (радиус 1.0) частица помечается для движения по траектории.
На GPU:
Для захваченных частиц позиция вычисляется по параметрическому уравнению кривой Безье.
Прогресс движения (параметр t) увеличивается каждый кадр.
При достижении t > 1.0 частица возвращается в свободный режим.
2.5. Геометрия и атрибуты частиц

Использовать THREE.BufferGeometry с атрибутами:
Начальные позиции (случайные координаты в заданной области).
Дополнительные атрибуты: направление базового движения, прогресс по сплайну, флаг захвата.
2.6. Пользовательский интерфейс

Элементы управления:
Две кнопки: «Аттрактор мыши» и «Сплайн-путь».
При клике на кнопку обновляется uniform-переменная режима в шейдере.
Требования к UX:
Мгновенное переключение режимов без видимых задержек.
Плавная анимация перехода между состояниями частиц.
3. Критерии успеха

Производительность:
60 FPS на MacBook Air M1 при 1000 частицах.
Отсутствие падений FPS на Android-устройствах среднего класса (Snapdragon 7-series).
Функциональность:
В режиме 1: мгновенная реакция частиц на движение курсора.
В режиме 2: плавное движение частиц по сплайну без рывков при захвате и выходе из траектории.
Стабильность:
Отсутствие утечек памяти при длительной работе и частом переключении режимов.
4. Дополнительные указания

Для расчёта расстояния до кривой Безье использовать аппроксимацию через ломаную линию (30–50 сегментов) для баланса производительности и точности.
При невозможности реализовать движение по сплайну полностью на GPU разрешается гибридный подход: базовое движение и отталкивание — на GPU, логика захвата и прогресса по пути — на CPU.
Для оптимизации использовать Instanced Rendering (THREE.InstancedMesh) или аналоги для минимизации draw call’ов.
Все uniform-переменные и атрибуты должны обновляться минимально необходимое количество раз за кадр.
